# 旧ツール出力 vs 新コンバーター出力 全件比較レポート

警察庁交通事故統計オープンデータについて、**旧ツール（複数リポジトリ）の変換出力**と
**新コンバーター（本リポジトリ）の変換出力**を全件突き合わせ、差異を確定・整理したもの。

- **実施日**: 2026-07-04
- **結論**: 値の差異は **2022年の3項目（5列）のみ**。2019・2020・2021・2023・2024は共通列で完全一致。
  差異はすべて **新コンバーターが公式コード表に準拠して旧ツールの欠落・誤りを補正した方向**。

---

## 1. 比較の対象と方法

### 対象（旧ツール）

| 旧リポジトリ | 対応年 | 出力形式 |
|---|---|---|
| `shiwaku/npa-traffic-accident-data-converter`（無印） | 2019・2020・2021 | 3年結合1ファイル・60列 |
| `shiwaku/npa-traffic-accident-data-2022-converter` | 2022 | 単年・72列 |
| `shiwaku/npa-traffic-accident-data-2023-converter` | 2023 | 単年・72列 |
| `shiwaku/npa-traffic-accident-data-2024-converter` | 2024 | 単年・72列 |

※ 2022の入手データが破損（重複）していたため、旧2022リポジトリのスクリプト（`to-degree`→`convert`）で
同一入力から再生成して比較した。旧repoスクリプト自体にバグはなく、再生成結果は正常（300,839行）。

### 方法

1. 旧・新とも同一の入力本票CSV（`data/{年}/honhyo_{年}.csv`）から生成された出力を使用。
2. 旧新は緯度経度の不正行スキップ条件が同一のため、**行の並びが一致** → 行位置で1対1に突き合わせ。
3. **本票番号列**を毎行照合して対応の正しさを検証（全年で不一致 0 件）。
4. 新旧に共通する列のみ値比較。列構成の差（新のみ/旧のみ列）は構造差として別途記録。

## 4. 構造差（2019-2021）

旧ツール（無印リポジトリ）は 60列、新コンバーターは全年 **72列の統一スキーマ**。
2019-2021では新側に以下 **12列** が追加されるが、元データに列が存在しないため**すべて空値**。

`日の出時刻 時/分`・`日の入り時刻 時/分`・`オートマチック車(A/B)`・`サポカー(A/B)`・
`認知機能検査経過日数(A/B)`・`運転練習の方法(A/B)`

これらは 2022年のファイル定義書改訂（60→72列）で追加された項目。値の差異ではなく列構成の差のため、
`check_output_diff.py` の `KNOWN_NEW_ONLY_COLS` として許容し、共通60列のみ比較している。

## 5. 検証（再現手順）

旧出力は `reference/` に取り込み済み。以下で全年 PASS することを確認：

```bash
python scripts/check_output_diff.py --all --known-diffs
# → 2019-2021: 構造列12件（許容）＋全行一致 / 2022: 意図的差異除外で一致 / 2023-2024: 完全一致
```

意図的差異は `scripts/check_output_diff.py` の `KNOWN_DIFFS`（2022の5パターン）に登録済み。

## 6. 補足・過去記載の訂正

本比較の結果、旧 `CHECKS.md` に「意図的差異」として記載されていた次の2件は**実在しない**ことが判明し削除した：

- エアバッグの装備「非展開 → その他」各年40〜50万件 … 旧4リポジトリともコード2は公式どおり「その他」を出力しており、**全年で差異なし**。
- 2023年 当事者種別 差異 約19,608件 … **2023は全307,930行が完全一致**。

また、旧2022の入手ファイル（`honhyo_2022_convert_BROKEN.csv`）は重複により行数が約2倍・ヘッダー混入があり、
比較不能だったため再生成品に差し替えた（破損ファイルは削除済み）。
