# 警察庁交通事故統計オープンデータ コンバーター

警察庁が公開する[交通事故統計情報のオープンデータ](https://www.npa.go.jp/publications/statistics/koutsuu/opendata/index_opendata.html)（本票CSV）をコード値から人間可読なコード値をラベルに変換し、緯度経度を10進数に変換するツールです。

2019〜2024年（令和元〜6年）対応。2025年以降のデータ追加にも対応した設計になっています。

---

## セットアップ

```bash
pip install -e .
```

依存ライブラリ: Python 3.10以上、pandas 2.0以上、openpyxl

---

## 使い方

### 1. データのダウンロード

```bash
# 全年次（2019〜2024）をダウンロード
./download.sh

# 特定年次のみ
./download.sh 2024
./download.sh 2022 2023 2024
```

`data/{year}/` に本票CSV・ファイル定義書・コード表xlsxが保存されます。

### 2. 変換の実行

```bash
# 単年
python -m converter --year 2024

# 複数年
python -m converter --year 2022 2023 2024

# 全既知年（2019〜2024）
python -m converter --all

# 全年次を1ファイルにマージ
python -m converter --all --merge
```

出力先: `output/honhyo_{year}_converted.csv`（UTF-8）

## 出力フォーマット

72列のCSV（UTF-8）。「種別」列の意味は以下の通りです：

- **コード値をラベルに変換**: 元データの数値コードを人間が読めるコード値をラベルに変換
- **元データのまま**: 元データの値を元データのまま出力
- **10進数変換**: 度分秒形式（例: `354119.5` = 北緯35度41分19.5秒）を10進数（例: `35.688...`）に変換
- **※空値**: 2019〜2021年は元データにこの列が存在しないため空値

全列一覧（`converter/convert.py` の `OUTPUT_FIELDNAMES` と同順）：

**事故基本情報**

| 列名 | 種別 |
|------|------|
| 資料区分 | コード値をラベルに変換（`1`=本票、`2`=補充票、`3`=高速票） |
| 都道府県名 | コード値をラベルに変換（例: `13`=東京都） |
| 警察署等名 | コード値をラベルに変換（都道府県コード＋署コード→署名） |
| 本票番号 | 元データのまま |
| 事故内容 | コード値をラベルに変換（`1`=死亡事故、`2`=負傷事故） |
| 死者数 | 元データのまま |
| 負傷者数 | 元データのまま |

**発生場所・路線**

| 列名 | 種別 |
|------|------|
| 路線名 | コード値をラベルに変換（路線コードから路線名を生成。高速・専用道は路線CSVを参照） |
| 上下線 | コード値をラベルに変換（`1`=上、`2`=下）。2022年以降は空値 |
| 地点コード | 元データのまま |
| 市区町村コード | 元データのまま |

**発生日時**

| 列名 | 種別 |
|------|------|
| 発生日時_年 / 月 / 日 / 時 / 分 | 元データのまま |
| 曜日(発生年月日) | コード値をラベルに変換（`1`=日曜日〜`7`=土曜日） |
| 祝日(発生年月日) | コード値をラベルに変換（`1`=祝日、`2`=祝日でない） |
| 日の出時刻　　時 / 分 | 元データのまま（※空値） |
| 日の入り時刻　　時 / 分 | 元データのまま（※空値） |

**道路状況**

| 列名 | 種別 |
|------|------|
| 昼夜 | コード値をラベルに変換（`1`=昼間、`2`=夜間など） |
| 天候 | コード値をラベルに変換（`1`=晴、`2`=曇、`3`=雨など） |
| 地形 | コード値をラベルに変換（`1`=市街地、`2`=郡部、`3`=高速道路等） |
| 路面状態 | コード値をラベルに変換（`1`=乾燥、`2`=湿潤など） |
| 道路形状 | コード値をラベルに変換（`01`=交差点、`02`=交差点付近など） |
| 環状交差点の直径 | コード値をラベルに変換。2019〜2021年は空値 |
| 信号機 | コード値をラベルに変換 |
| 一時停止規制_標識（当事者A） | コード値をラベルに変換 |
| 一時停止規制_表示（当事者A） | コード値をラベルに変換 |
| 一時停止規制_標識（当事者B） | コード値をラベルに変換 |
| 一時停止規制_表示（当事者B） | コード値をラベルに変換 |
| 車道幅員 | コード値をラベルに変換（例: `01`=単路－3.5m未満） |
| 道路線形 | コード値をラベルに変換（例: `9`=直線－平坦） |
| 衝突地点 | コード値をラベルに変換（`01`=単路、`30`=交差点内など） |
| ゾーン規制 | コード値をラベルに変換（`01`=ゾーン30など） |
| 中央分離帯施設等 | コード値をラベルに変換 |
| 歩車道区分 | コード値をラベルに変換 |
| 事故類型 | コード値をラベルに変換（例: `01`=人対車両－横断中） |

**当事者情報（当事者A・当事者Bそれぞれ）**

| 列名 | 種別 |
|------|------|
| 年齢（当事者A/B） | コード値をラベルに変換（例: `04`=20〜24歳） |
| 当事者種別（当事者A/B） | コード値をラベルに変換（例: `03`=乗用車－普通車、`61`=歩行者） |
| 用途別（当事者A/B） | コード値をラベルに変換 |
| 車両形状（当事者A/B） | コード値をラベルに変換（例: `01`=乗用車、`43`=特定小型原付自転車） |
| 速度規制（指定のみ）（当事者A/B） | コード値をラベルに変換（例: `40`=40km/h） |
| 車両の衝突部位（当事者A/B） | コード値をラベルに変換 |
| 車両の損壊程度（当事者A/B） | コード値をラベルに変換 |
| エアバッグの装備（当事者A/B） | コード値をラベルに変換 |
| サイドエアバッグの装備（当事者A/B） | コード値をラベルに変換 |
| 人身損傷程度（当事者A/B） | コード値をラベルに変換（`1`=死亡、`2`=重傷、`3`=軽傷） |
| オートマチック車（当事者A/B） | 元データのまま（※空値） |
| サポカー（当事者A/B） | 元データのまま（※空値） |
| 認知機能検査経過日数（当事者A/B） | 元データのまま（※空値） |
| 運転練習の方法（当事者A/B） | 元データのまま（※空値） |

**緯度経度**

| 列名 | 種別 |
|------|------|
| 地点_緯度（北緯） | 元データのまま（度分秒形式、例: `354119.5`） |
| 地点_経度（東経） | 元データのまま（度分秒形式、例: `1394459.5`） |
| 地点_緯度（北緯）_10進数 | 10進数変換（例: `35.688...`） |
| 地点_経度（東経）_10進数 | 10進数変換（例: `139.749...`） |

## 年次別コード表の補足

| 年次 | コード辞書 | CSV（警察署等・路線・トンネル） |
|------|-----------|-------------------------------|
| 2019〜2021 | `codes/y2019_2021.py` | `code_tables/2019-2021/` |
| 2022 | `codes/y2022.py` | `code_tables/2022/` |
| 2023 | `codes/y2023.py` | `code_tables/2023/` |
| 2024 | `codes/y2024.py` | `code_tables/2024/` |

警察庁が公開するコード表は年次によって内容が異なります。このコンバーターでは、年次ごとに異なるコード辞書（`converter/codes/yYYYY.py`）とCSVファイル（`code_tables/{year}/`）を使い分けています。

### 年次ごとの公式コード表の変更点

#### 2022年（令和4年）の変更
公式ファイル定義書が改訂され、本票の出力カラムが **60列→72列** に増えました。追加された主なカラム：

| 追加カラム | 内容 |
|-----------|------|
| 日の出時刻 時・分 | 発生日の日の出時刻 |
| 日の入り時刻 時・分 | 発生日の日の入り時刻 |
| オートマチック車（当事者A/B） | AT車かMT車か |
| サポカー（当事者A/B） | サポートカーの装備有無 |
| 認知機能検査経過日数（当事者A/B） | 高齢運転者の検査経過日数 |
| 運転練習の方法（当事者A/B） | 路上練習・場内練習など |

2019〜2021年の出力CSVでは、これら12カラムは空値になります。

#### 2024年（令和6年）の変更
公式コード表（当事者種別）に **電動キックボード等の新区分** が追加されました：

| コード | 2023年まで | 2024年以降 |
|--------|-----------|-----------|
| `36` | `二輪車－原付自転車` | `二輪車－一般原付自転車`（表現変更） |
| `43` | （なし） | `特定小型原付自転車`（新規追加・電動キックボード等） |

#### 警察署等コード・高速路線コード（毎年更新）
警察署の統廃合や高速道路の新規開通に伴い、警察署等コード・高速路線コードは毎年更新されます。このため `code_tables/2022/`・`2023/`・`2024/` はそれぞれ異なるCSVファイルを使用しています。

#### 2019〜2021年の警察署等・高速路線について
2019〜2021年用のCSV（`code_tables/2019-2021/`）は2019年版のみ収録されています。2020年・2021年に新設・変更された警察署や高速路線は、このCSVに含まれていません。該当するレコードは警察署等名・路線名が空欄で出力されます（将来の改善課題）。

#### 2024年のトンネル番号について
2024年の公式コード表（xlsx）にはトンネルの追加・変更が記載されています。ただし、変換に使用するトンネル番号CSV（`code_tables/2024/`）は2023年版から変更していないため、2024年に新設・変更されたトンネルについてはトンネル名が正確に出力されない場合があります。
